Azure Entra ID Setup Guide (Delegated / MSAL)
⚠ Required Azure permission: Mail.Send (Delegated Permission) + register this page URL as a Redirect URI (Single-page application)
  1. Sign in to portal.azure.com → Microsoft Entra ID → App Registrations → New Registration
  2. Name the app (e.g. SecureReport Mailer), choose your tenant, click Register
  3. Go to Authentication → Add a platform → Single-page application (SPA) → paste this page's full URL as the Redirect URI → Save
  4. Go to API Permissions → Add Permission → Microsoft Graph → Delegated Permissions → search Mail.Send → Add → Grant Admin Consent
  5. From Overview, copy the Application (client) ID and Directory (tenant) ID
  6. Paste the IDs above, enter the sender email, click Save Configuration
  7. Click Sign In with Microsoft — authenticate once. Token is cached; all future emails send silently.
  8. Click Send Test Email to verify ✓
🔒 Security note: MSAL delegated flow uses a user popup login — no client secret is stored in the browser. This is inherently safer than the client credentials flow. Tokens are cached in localStorage and refreshed silently.
