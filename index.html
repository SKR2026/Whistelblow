🚀 Setup — License System for SecureReport
Step 1 — Firebase Project
The app uses Firebase project skr-task.
The license gate reads from the licenses collection in the same project.
Firebase config is already set in this developer panel file.
Step 2 — Set Firestore Rules
Go to Firebase Console → skr-task → Firestore Database → Rules, select all, paste below and click Publish:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
⚠ Also enable Anonymous Auth: Firebase Console → Authentication → Sign-in providers → Anonymous → Enable.
Step 3 — How the Gate Works
• When a SuperAdmin logs in, the app checks for a cached license key in localStorage.
• If no cached key (or it expired/revoked), the License Gate overlay appears.
• The key is validated against the licenses Firestore collection.
• On success the key is cached; on Lock/logout the cache is cleared.
• Developer account bypasses the gate entirely.
• Other roles (Admin, Top Mgmt, SIO) never see the gate.
Step 4 — Files
• SecureReport_App.html → Deploy to server (public access)
• SIO_License_Manager.html → Keep on your machine ONLY, never publish
✓ SuperAdmins must have a valid license key before accessing the dashboard.
🔑 License Key Format
Keys generated as: XXX-XXXX-XXXX-XXXX
Characters: A-Z, 0-9 — excluding ambiguous chars (O, 0, I, 1)
Firestore collection: licenses · Document ID = license key (uppercase)
Fields: licenseKey, company, customerName, email, issuedDate, expiryDate, isActive, activationCount, notes, createdAt
